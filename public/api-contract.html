接口契约轻量文档 · MiniApp 与后端对齐

环境与基准

说明：接口基于当前项目配置文件（clients/miniapp/src/miniprogram/config/api.js 与 config/database.php）。如切换环境，请同步更新前端环境与后端域名。

参考：api 基础地址通过 deriveBaseUrl 提取；WebSocket 地址由 wsUrl 提供。

设备接口

GET device/profitconfig
获取设备分润配置
响应示例
{
  "code": 1,
  "data": {
    "device_id": 10086,
    "tiers": [
      { "tier_id": 1, "price": 2.00, "template": "basic", "profit_json": "{...}" }
    ]
  },
  "msg": "ok"
}

支付接口

POST pay/notify
支付结果通知
响应示例
{
  "code": 1,
  "data": { "order_id": 123456, "status": "paid" },
  "msg": "success"
}
POST pay/create
创建支付订单
{
  "code": 1,
  "data": { "order_id": 123456, "amount": 2.00, "pay_params": { /* prepay info */ } },
  "msg": "ok"
}
POST pay/query
查询并同步订单支付状态
{
  "code": 1,
  "data": { "order_id": 123456, "status": "paid" },
  "msg": "ok"
}

抽奖接口

POST device/smartLottery
智能抽奖：请求 { device_id, tier_id }
响应示例
{
  "code": 1,
  "data": {
    "record_id": 98765,
    "result": "hit",
    "prize_type": "physical",
    "goods_id": 1001,
    "amount": 2.00
  },
  "msg": "ok"
}
GET lottery/result
获取抽奖结果
{
  "code": 1,
  "data": { "record_id": 98765, "result": "hit", "order_id": 123456, "ext": { /* 扩展 */ } },
  "msg": "ok"
}
POST addons/turntable/api/turntable/draw
Turntable 插件抽一次
{
  "code": 1,
  "data": { "record_id": 555, "slot_id": 8, "goods_id": 1001 },
  "msg": "ok"
}
GET addons/turntable/api/turntable/record
抽奖记录查询

资源与分享

GET device/profitprizes
指定金额奖品列表
GET device/shareContent
获取分享内容（文案与图片）

WebSocket 契约

WS wsUrl (来自 config.api.js)
消息类型：lottery_result | notification | system_message
// 心跳（前端）
{ "type": "heartbeat", "ts": 1710000000 }

// 抽奖结果（服务端 -> 客户端）
{
  "type": "lottery_result",
  "payload": {
    "record_id": 98765,
    "result": "hit",
    "goods_id": 1001,
    "amount": 2.00
  }
}

// 通知消息（服务端 -> 客户端）
{ "type": "notification", "payload": { "level": "info", "text": "欢迎体验" } }
断线重连策略：指数退避，心跳看门狗（建议 30s 超时），手动断开不触发自动重连。

统一响应格式

{
  "code": 1,          // 成功=1，失败=0
  "data": { ... },    // 业务数据
  "msg": "ok"        // 描述信息
}
// 额外：401 表示 Token 过期（参考 config.api.js responseFormat）
若需补充，请联动前端 config/api.js 与后端控制器路由文档。
